# Файловий менеджер: Основні моделі (Python)

Цей репозиторій містить реалізацію базових класів для системи управління файлами (активами) творчих проєктів. Проєкт створено для демонстрації застосування принципів об'єктно-орієнтованого програмування (ООП) на мові Python.

##  Вимоги та реалізовані принципи ООП

У проєкті виконані ключові вимоги практичної роботи:

1. **Створення основних класів:** Реалізовано базову модель активу та спеціалізовані типи файлів (відео, код).
2. **Абстракція (`IAsset`):** Використано модуль `abc` для створення абстрактного базового класу (інтерфейсу), який гарантує наявність обов'язкового методу `get_info()` у всіх спадкоємців.
3. **Інкапсуляція (`BaseAsset`):** Стан об'єктів (назва, розмір, статус) захищено від прямого втручання за допомогою приватних атрибутів (наприклад, `__status`). Доступ та зміна відбуваються виключно через властивості (`@property` та `@status.setter`) із валідацією типів.
4. **Наслідування (`VideoAsset`, `CodeAsset`):** Спеціалізовані класи успадковують загальну логіку від `BaseAsset` та розширюють її унікальними атрибутами (тривалість для відео, мова для коду).
5. **Поліморфізм:** Перевизначення (overriding) методу `get_info()` у дочірніх класах із викликом батьківського методу через `super()`.
6. **Документація:** Код повністю покритий Docstrings (опис класів, методів та параметрів) для полегшення розуміння логіки.

## Структура проєкту

* `models.py` — містить основну логіку, перелік станів (`Enum`), абстракції та реалізацію класів.
* `test_models.py` — містить автоматизовані unit-тести для перевірки інкапсуляції, наслідування та коректності зміни станів.
