## Реалізація шаблону Singleton у Python

**Мета:** навчитися проєктувати та реалізовувати шаблон Singleton для гарантії єдиного екземпляра класу у програмі.

---

## 1. Теоретичні відомості

### Що таке Singleton
`Singleton` — це породжувальний шаблон проєктування, який гарантує:
- існування лише одного екземпляра класу;
- єдину точку доступу до цього екземпляра.

### Коли застосовують
- логування (єдиний logger/конфіг);
- робота з конфігурацією застосунку;
- кеш або пул з'єднань;
- доступ до спільного ресурсу (файл, сервіс, черга).

## 3. Висновки

- Шаблон `Singleton` зручний для централізованого доступу до спільних ресурсів.
- Лінива ініціалізація економить ресурси, бо об'єкт створюється лише за потреби.
- У багатопотокових задачах потрібна синхронізація (блокування), інакше можуть створитися кілька екземплярів.
- Singleton корисний, але його слід використовувати обережно через ризики жорсткої зв'язності та складності тестування.
